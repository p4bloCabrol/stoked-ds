# stoked-ds — Accessibility Checklist

Every component MUST implement all applicable items.

## ARIA

- [ ] Correct ARIA role (`role="dialog"`, `role="button"`, `role="region"`, etc.)
- [ ] `aria-expanded` for collapsible content
- [ ] `aria-controls` linking trigger to content panel
- [ ] `aria-label` or `aria-labelledby` for non-text elements
- [ ] `aria-disabled` for disabled states (in addition to native `disabled` attribute)
- [ ] `aria-busy` for loading states
- [ ] `aria-hidden="true"` on decorative icons and elements
- [ ] Unique IDs via `useId()` for `aria-controls` / `aria-labelledby` linkage

## Keyboard navigation

- [ ] Enter / Space → activate buttons and controls
- [ ] Escape → close modals, dropdowns, tooltips
- [ ] Arrow keys → navigate lists, menus, tabs, trees
- [ ] Tab / Shift+Tab → move between focusable elements
- [ ] Focus trap for modals (`tabIndex`, focus management on open/close)

## Focus styles

```css
.root:focus-visible {
  outline: 2px solid var(--stoked-color-focus-ring);
  outline-offset: 2px;
}
```

Never suppress focus styles. Use `:focus-visible` (not `:focus`) to avoid showing outlines on mouse click.

## Motion

```css
@media (prefers-reduced-motion: reduce) {
  .root { transition: none; animation: none; }
}
```

Always include this block in any component that has `transition` or `animation`.

## Semantic HTML

- Use native elements when possible (`<button>`, `<input>`, `<select>`, `<nav>`, `<main>`)
- Only use `role` when a native element isn't available or appropriate
- Don't use `div` or `span` as interactive elements without proper `role` + keyboard support
